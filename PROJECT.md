# ASA Fan Website

Static marketing + product-catalog site for **ASA Fan** (Adda South America Corporation), a Brazilian manufacturer of microventiladores AC/DC, ventiladores axiais, and acessórios (telas, grelhas, porta-filtros, termostatos). Founded 1999, based in Cotia/SP.

- **Live:** https://eulahwu917.github.io/asafan-website/
- **Hosting:** GitHub Pages (auto-deploys from `main`)
- **Stack:** Hand-written HTML / CSS / vanilla JS — no build step, no framework, no package.json
- **Fonts:** Inter via Google Fonts. **Icons:** Lucide (CDN). **Color tokens:** navy `#1A1F60`, red `#CC0000`.

## Pages

| File | Purpose |
|---|---|
| [index.html](index.html) | Homepage — hero video, portfolio cards, value props, 3-step process, CTA |

| [produtos.html](produtos.html) | **Product catalog** — 77 cards, two-tier tipo filter + contextual secondary filters. Pre-selects via `?tipo=micro-ac\|micro-dc\|axial\|acessorio`. Cards link to per-product detail pages. |
| [produto/](produto/) | **Per-product detail pages** — one static page per SKU (77 files, named `<slug>.html`). Auto-generated by `scripts/generate_cards.py`. | [aplicacoes.html](aplicacoes.html) | 6 application cards (compressores, painéis elétricos, elevadores, automação, peltier, equipamentos médicos) |

All pages share the same header (with Produtos dropdown) and 4-column footer. Every CTA links to `https://wa.me/551134065088` with a pre-filled message.

## Product catalog data model

Source of truth: **`raw_assets/product_site/dados para site.xlsx`** (sheet `Planilha1`, ~89 rows). Run **`python scripts/generate_cards.py --apply`** to regenerate everything from the xlsx in one shot:

1. Catalog cards in [produtos.html](produtos.html) (between the `BEGIN: product cards` / `END: product cards` HTML comment markers — idempotent re-runs).
2. Per-product detail pages in [produto/](produto/) — one `<slug>.html` per SKU.
3. Product URL block in [sitemap.xml](sitemap.xml) (between `BEGIN/END: product detail pages` markers).

77 products grouped by SKU prefix:

| Prefix | Tipo | Count |
|---|---|---|
| `A`, `E` | Microventilador AC (110V/220V/bivolt) | 10 |
| `D` | Microventilador DC (12V/24V/48V) | 30 |
| `X` | Axial (soprador/exaustor, 220V mono / 220V-380V tri) | 21 |
| `M` | Telas/máscaras metálicas | 7 |
| `G` | Grelhas ABS | 3 |
| `P` | Porta-filtros | 3 |
| `T` | Termostatos | 3 |

**Filter conventions** (data-attributes on `<article class="product-card">`):
- `data-tipo` — `micro-ac` | `micro-dc` | `axial` | `acessorio`
- `data-subtipo` — `tela` | `grelha` | `porta-filtro` | `termostato` (only on acessórios)
- `data-tensao` — `12v` | `24v` | `48v` | `110v` | `220v` | `bivolt` | `220v-380v`
- `data-mancal` — `rolamento` | `bucha` (microventiladores)
- `data-funcao` — `soprador` | `exaustor` (axiais)

**Contextual filter visibility** — `.filter-group[data-context="..."]` lists which `tipo` values reveal that group; the JS in [js/main.js](js/main.js) hides non-matching groups and ignores their values when filtering. Two separate Tensão groups exist (DC vs Axial); active state is scoped to the parent `.filter-group` to keep them independent.

**Image lookup** — each card's `<img src>` points to `assets/product/<código-lowercase>.jpg|png`; missing photos fall back to `assets/product/placeholder.png` ("Foto em breve").

## Conventions

- **Company name:** "ASA Fan" everywhere (not ASAFAN, AsaFan, etc.).
- **Image filenames:** lowercase código (`a81.jpg`, `d91-2.jpg`) — required because GitHub Pages is case-sensitive.
- **No build pipeline** — edit HTML/CSS/JS directly; commit; push to `main` → Pages deploys.
- **Forms:** all use `mailto:` (static-host limitation). Replacement is on the deferred list.
- **External links:** `target="_blank" rel="noopener noreferrer"`.
- **Accessibility:** form `<label for>`/`<input id>` paired everywhere; nav has `aria-label`; dropdown supports `:focus-within`.
- **Structured data:** Organization JSON-LD on every page; BreadcrumbList where applicable. `sameAs` is empty (TODO).
- **Telemetry/analytics:** none configured.

## Customer-supplied source data

- **`raw_assets/product_site/dados para site.xlsx`** — canonical product spreadsheet (do not infer specs from the rendered HTML; regenerate via `generate_cards.py`).
- **`raw_assets/product_site/FOTOS ATUAIS_/`** — original studio photos (38 files, named by código).
- **`raw_assets/product_site_update/`** — 2026-05-04 photo drop. Subfolders: `AC/ DC/ AXIAIS/ ACESSÓRIOS/ FOTOS ATUAIS_/`. Routing convention used by [scripts/integrate_photos.py](scripts/integrate_photos.py): `FOTOS ATUAIS_/` is canonical for AC/DC solo photos; `AXIAIS/` and `ACESSÓRIOS/` are the only sources for those categories. Files referencing only manufacturer model numbers (not in xlsx CÓDIGO) are ignored.
